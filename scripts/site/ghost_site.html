SD Card Pin Configuration

Note: The SD Card communication mode (SDMMC or SPI) is determined by the firmware's compile-time configuration (sdkconfig). This section allows setting the GPIO pins for *both* modes, but only the pins corresponding to the *compiled mode* will be used. Changing the actual mode requires recompiling and flashing new firmware. Setting pins for an inactive mode will have no effect.

ESP32 Commands

  • help: Display this help message.
    Usage: help
  • scanap: Start a Wi-Fi access point (AP) scan.
    Usage: scanap
  • scansta: Start scanning for Wi-Fi stations.
    Usage: scansta
  • stopscan: Stop any ongoing Wi-Fi scan.
    Usage: stopscan
  • attack: Launch an attack (e.g., deauthentication attack).
    Usage: attack -d
    Arguments:
    -d: Start deauth attack
  • list: List Wi-Fi scan results or connected stations.
    Usage: list -a | list -s
    Arguments:
    -a: Show access points from Wi-Fi scan
    -s: List connected stations
  • beaconspam: Start beacon spam with different modes.
    Usage: beaconspam [OPTION]
    Arguments:
    -r: Start random beacon spam
    -rr: Start Rickroll beacon spam
    -l: Start AP List beacon spam
    [SSID]: Use specified SSID for beacon spam
  • stopspam: Stop ongoing beacon spam.
    Usage: stopspam
  • stopdeauth: Stop ongoing deauthentication attack.
    Usage: stopdeauth
  • select: Select an access point by index from scan results.
    Usage: select -a <number>
    Arguments:
    -a: AP selection index (must be a valid number)
  • startportal: Start an Evil Portal.
    Usage: startportal <FilePath> <AP_SSID> <PSK>
    Arguments:
    <FilePath>: Local file path (e.g., index.html)
    <AP_SSID>: SSID for the access point
    <PSK>: PSK for the access point
  • stopportal: Stop the Evil Portal.
    Usage: stopportal
  • blescan: Handle BLE scanning with various modes.
    Usage: blescan [OPTION]
    Arguments:
    -f: Start 'Find the Flippers' mode
    -ds: Start BLE spam detector
    -a: Start AirTag scanner
    -r: Scan for raw BLE packets
    -s: Stop BLE scanning
  • capture: Start a Wi-Fi capture (requires SD card or Flipper).
    Usage: capture [OPTION]
    Arguments:
    -probe: Start capturing probe packets
    -beacon: Start capturing beacon packets
    -deauth: Start capturing deauth packets
    -raw: Start capturing raw packets
    -wps: Start capturing WPS packets and their auth type
    -pwn: Start capturing Pwnagotchi packets
    -stop: Stop the active capture
  • connect: Connect to a specific Wi-Fi network.
    Usage: connect <SSID> <Password>
  • dialconnect: Cast a random YouTube video on all smart TVs on LAN.
    Usage: dialconnect
  • powerprinter: Print custom text to a printer on LAN.
    Usage: powerprinter <Printer IP> <Text> <FontSize> <Alignment>
    Alignment options: CM = Center Middle, TL = Top Left, TR = Top Right, BR = Bottom Right, BL = Bottom Left
  • blewardriving: Start/Stop BLE wardriving with GPS logging.
    Usage: blewardriving [-s]
    Arguments:
    -s: Stop BLE wardriving
  • scanports: Scan ports on local subnet or specific IP.
    Usage: scanports local [-C/-A/start_port-end_port]
    scanports [IP] [-C/-A/start_port-end_port]
    Arguments:
    -C: Scan common ports only
    -A: Scan all ports (1-65535)
    start_port-end_port: Custom port range (e.g. 80-443)
  • setrgbpins: Change RGB LED pins configuration.
    Usage: setrgbpins <red> <green> <blue>
    Note: Use same value for all pins for single-pin LED strips
